## 🔴 严重问题

### 问题 1: 内容区域过度限制导致空间浪费

**位置**: `Studio.tsx:514`

```tsx
<main
  style={{
    maxWidth: "1200px",
    margin: "0 auto",
    padding: "24px",
    gap: "32px",
  }}
>
```

**问题分析**:
- 在 27" 显示器（2560px 宽）上，内容仅占 47% 宽度
- 两侧留白过大，视频舞台显得局促
- `maxWidth: 1200px` 对于视频创作工具过于保守

**视觉表现**:
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌───────────────────────────────────────────────────┐     │
│   │                                                   │     │
│   │              1200px 内容区域                       │     │
│   │          (仅占屏幕宽度的 47%)                      │     │
│   │                                                   │     │
│   │    ┌─────────────────────────────┐                │     │
│   │    │      视频舞台 (960px)        │                │     │
│   │    │                            │                │     │
│   │    └─────────────────────────────┘                │     │
│   │                                                   │     │
│   └───────────────────────────────────────────────────┘     │
│                                                             │
│         ↑ 大量浪费空间 ↑               ↑ 大量浪费空间 ↑       │
└─────────────────────────────────────────────────────────────┘
```

**修复建议**:
```tsx
// 方案 1: 增加最大宽度
maxWidth: "1600px",

// 方案 2: 根据屏幕尺寸动态调整
maxWidth: "min(90vw, 1400px)",

// 方案 3: 视频舞台全宽，其他内容限制
// 视频舞台: maxWidth: "1200px" 或 100%
// 输入框: maxWidth: "800px" 保持适中
```

### 问题 2: 视频舞台尺寸与容器不匹配

**位置**: `VideoStage.tsx:389-399`

```tsx
style={{
  maxWidth: "960px",
  aspectRatio: "16 / 9",
  minHeight: "280px",
  maxHeight: "70vh",
}}
```

**问题分析**:
1. `maxWidth: 960px` 但容器 `maxWidth: 1200px`，比例不协调
2. `maxHeight: 70vh` 在大屏幕上可能仍显小
3. 装饰边框 `inset: 6px` 导致实际视频区域更小

**实际可用空间计算**:
```
视频舞台总宽度: 960px
- 装饰边框 (左右各 6px): -12px
- 内容区域内边距 (左右各 12px): -24px
实际视频宽度: 924px

对于 1080p 视频 (1920x1080):
显示比例: 924/1920 = 48%
严重缩小，细节难以看清
```

**修复建议**:
```tsx
// VideoStage.tsx
style={{
  width: "100%",
  maxWidth: "min(100%, 1200px)", // 与容器同宽或更大
  aspectRatio: "16 / 9",
  maxHeight: "min(80vh, 800px)", // 增加高度限制
}}

// 或者根据分辨率动态调整
const getStageMaxWidth = () => {
  if (window.innerWidth >= 1920) return "1400px";
  if (window.innerWidth >= 1440) return "1200px";
  return "960px";
};
```

### 问题 3: 输入框与视频舞台宽度不一致导致视觉错位

**位置**: 
- `VideoStage.tsx:389`: `maxWidth: "960px"`
- `ChatInput.tsx:136`: `maxWidth: "800px"`

**问题分析**:
- 视频舞台 960px，输入框 800px
- 两者都居中，但宽度不同，视觉上产生"断层"
- 用户视线需要在两个不同宽度间切换

**视觉问题**:
```
┌──────────────────────────────────────┐  ← 视频舞台 (960px)
│                                      │
│              视频内容                 │
│                                      │
└──────────────────────────────────────┘
        ↓ 视觉断层 ↓
    ┌──────────────────────────┐      ← 输入框 (800px)
    │   输入动画描述...     [➤] │
    └──────────────────────────┘
```

**修复建议**:
```tsx
// 方案 1: 统一宽度
// VideoStage 和 ChatInput 都使用相同的 maxWidth
maxWidth: "800px", // 与输入框一致

// 方案 2: 输入框更宽以匹配视频
maxWidth: "960px", // 与视频舞台一致

// 方案 3: 全宽设计，通过内部约束控制
width: "100%",
// 内部元素使用 padding 控制实际宽度
```

### 问题 4: 工具栏按钮过多且"帮助"按钮无功能

**位置**: `Studio.tsx:588-604`

```tsx
<ToolbarButton icon={<History size={20} />} label="历史" />
<ToolbarButton icon={<Settings size={20} />} label="设置" />
<ToolbarButton icon={<HelpCircle size={20} />} label="帮助" onClick={() => {}} />
<div style={{ width: "8px" }} />
<ThemeToggle />
```

**问题分析**:
1. **"帮助"按钮无功能**: `onClick={() => {}}` 是空函数
2. **4 个控件拥挤**: 在紧凑 Header 中显得杂乱
3. **无优先级区分**: 历史和设置是核心功能，帮助是次要的

**修复建议**:
```tsx
// 方案 1: 移除无功能的帮助按钮
// 快捷键 ? 已足够

// 方案 2: 将帮助放入设置面板
// 设置面板中添加"快捷键"选项卡

// 方案 3: 工具栏分组
<div style={{ display: "flex", gap: "8px" }}>
  {/* 核心功能 */}
  <ToolbarButton icon={<History />} label="历史" />
  <ToolbarButton icon={<Settings />} label="设置" />
</div>
<div style={{ width: "1px", height: "24px", background: "var(--border-subtle)" }} />
<div style={{ display: "flex", gap: "8px" }}>
  {/* 辅助功能 */}
  <ThemeToggle />
</div>
```

### 问题 9: 间距系统不统一

**审计发现的各种间距值**:
- Header padding: `8px 16px`
- Main gap: `32px`
- ChatInput margin: `20px`
- VideoStage inset: `12px`
- ToolbarButton padding: `8px 14px`

**问题**:
- 缺乏统一的间距系统
- 4px, 6px, 8px, 12px, 14px, 16px, 20px, 32px 混用
- 视觉节奏不一致

**建议**: 建立 4px 基数系统
```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-6: 24px;
--space-8: 32px;
```

## 📐 布局结构分析

### 当前布局层次

```
studio (min-height: 100vh, flex, overflow: hidden)
├── SkyBackground (absolute, 全屏)
└── main (flex: 1, maxWidth: 1200px, margin: auto)
    ├── header (flex, space-between)
    │   ├── Logo
    │   └── Toolbar
    ├── ErrorBanner (conditional)
    ├── VideoStage (flex: 1, maxWidth: 960px)
    └── ChatInput (maxWidth: 800px)
```

**问题**:
1. `overflow: hidden` 在 studio 可能导致内容截断
2. `main` 的 `flex: 1` 与 VideoStage 的 `flex: 1` 冲突
3. 多层 maxWidth 限制导致空间浪费

## ✅ 总结

当前电脑端 UI 存在**空间利用率低**和**视觉对齐问题**：

1. **过度约束的 maxWidth** 导致大屏浪费空间
2. **视频与输入框宽度不匹配** 造成视觉断层
3. **装饰元素过多** 分散注意力
4. **间距系统混乱** 缺乏节奏感

**核心建议**: 
- 放宽内容宽度限制至 1400px+
- 统一视频和输入框宽度
- 移除不必要的装饰元素
- 建立统一的间距系统
